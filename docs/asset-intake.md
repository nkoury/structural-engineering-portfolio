# Asset Intake Guide

Use this guide when preparing a project for the portfolio.

## First Project Inputs

- Public-safe project title or alias
- Project type, year, region, and your role
- Structural system and project scale
- Engineering challenge and constraints
- What you personally modeled, calculated, detailed, reviewed, or coordinated
- Public-safe outcome or design takeaway

## Asset Categories

- `model`: U3D PDF references, GLB/GLTF exports, model screenshots, turntable videos
- `drawings`: plan crops, section crops, detail crops, redline/markup excerpts
- `analysis`: FEA contours, load path diagrams, demand summaries, reaction checks
- `calculations`: hand calc excerpts, design summaries, selected verification checks
- `field`: site photos, observation sketches, existing-condition documentation

## Redaction Checklist

- Client, owner, architect, and contractor names
- Exact address and site identifiers
- Professional stamps, signatures, license numbers, and seals
- Proprietary calculation pages or model metadata
- Full drawing sets unless intentionally released
- Any project notes that reveal confidential business, budget, schedule, or claim details

## Preferred Web Formats

- Images: `.webp`, `.jpg`, `.png`
- Interactive models: `.glb`, `.gltf`
- Video clips: `.mp4`, `.webm`
- Public documents: redacted `.pdf`
- U3D workflow references: redacted U3D-containing PDFs, usually linked rather than embedded
